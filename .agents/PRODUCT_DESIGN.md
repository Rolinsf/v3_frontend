# 若林轻小说：产品页面与视觉设计规划

本文档定义“若林轻小说”前台、阅读器和创作中心的页面范围、信息架构、布局、交互与视觉风格。后续设计和开发以此为基准。

## 1. 产品定位

若林轻小说是面向国内小圈子用户的轻小说阅读与创作社区。公开作品以开放阅读为原则，用户无需注册或登录即可浏览作品与阅读公开章节。核心体验不是信息轰炸，而是：

- 安静、舒适地发现与阅读作品；
- 低门槛地投稿、整理卷章和持续创作；
- 围绕作品与章节进行克制、友善的交流；
- 让收藏、进度和更新提醒简单可靠。

产品关键词：

```text
淡雅 / 安静 / 清晰 / 温和 / 文学感 / 轻社区 / 轻微二次元 / 移动端友好
```

明确避免：大红大紫、高饱和渐变、霓虹光效、过重投影、密集排行榜、弹窗轰炸、后台管理模板感和过度二次元装饰。允许使用少量线稿、纸片、星点、树叶、书签或角色剪影营造轻小说气质，但不能降低信息可读性。

## 2. 品牌与视觉方向

### 品牌名

- 中文名：若林轻小说
- 简称：若林
- 英文辅助标识：Wakabayashi Light Novels（只用于品牌辅助，不作为主标题）
- 品牌意象：林间纸页、嫩叶、晨雾、木色书签

Logo 第一阶段使用“若林”二字字标配一枚极简叶片/书页符号，不使用复杂人物插画作为主 Logo。

### 轻微二次元装饰

二次元元素用于营造气氛，不承担核心信息或交互：

- 首页欢迎区可以使用一张低饱和角色半身线稿、林间剪影或书页插画；
- 空状态可以使用简洁的读书、写作、找书小插图；
- 区块标题可搭配星点、叶片、墨水笔或书签小图标；
- 创作中心可使用纸张、羽毛笔、便签等轻量装饰；
- 阅读器只允许极淡的章节分隔花饰，不放人物背景和动态装饰；
- 装饰图透明度保持克制，不能覆盖文字，移动端可进一步弱化或隐藏；
- 一个视区只保留一个主要插画焦点，不同时堆叠角色、纹理、渐变和大量图标；
- 插画色彩从品牌色板取色，以鼠尾草绿、暖灰、木色为主。

角色插画应作为可替换品牌素材管理，不把具体角色直接耦合进页面结构。

## 3. 登录与权限原则

### 无需登录

- 浏览首页、书库、排行和搜索；
- 查看小说详情、作者公开主页、卷章目录；
- 阅读所有公开章节；
- 查看作品评论和章节评论；
- 使用本地阅读主题、字号、行高和字体设置；
- 在当前浏览器保存本地阅读进度。

阅读入口不得被登录弹窗、强制注册遮罩或“阅读若干章后登录”拦截。

### 操作时要求登录

- 点赞、评论、回复和举报；
- 收藏作品、同步书架和跨设备阅读进度；
- 接收通知；
- 发表、编辑和管理自己的小说、卷、章节及草稿；
- 管理自己的个人资料、账户安全和自己发表的评论。

访客触发需要身份的操作时，才显示轻量登录引导。登录成功后返回原页面，并在安全可行时继续原操作。

### 管理员权限

- `/admin/**` 只允许管理员进入；
- 普通用户即使是作者，也不能进入管理后台；
- 管理员身份和权限必须由后端验证，前端中间件只负责页面体验；
- 管理后台只包含全站内容审核、举报处置、用户治理、分类标签配置和系统级管理；
- 用户自己的作品、资料和评论管理全部属于前台用户空间，不放进管理后台。

## 4. 全站信息架构

### 主色板

整体使用低饱和鼠尾草绿、暖白和墨灰。

| Token | 色值 | 用途 |
| --- | --- | --- |
| `brand-50` | `#F4F7F2` | 淡绿色背景 |
| `brand-100` | `#E6EEE2` | 选中项和浅提示 |
| `brand-300` | `#AFC2A8` | 边框、次级装饰 |
| `brand-500` | `#6F8F68` | 主按钮、链接、品牌标识 |
| `brand-600` | `#5D7B57` | 主按钮悬停 |
| `brand-700` | `#4A6546` | 强调文字 |
| `paper` | `#FBFAF7` | 页面主背景，避免纯白刺眼 |
| `surface` | `#FFFFFF` | 卡片和浮层 |
| `ink` | `#2F332E` | 主文字 |
| `muted` | `#747A72` | 次要文字 |
| `line` | `#E6E8E2` | 分隔线和卡片边框 |
| `warm` | `#B4875F` | 少量书签/更新辅助色 |
| `danger` | `#B65F5A` | 删除和错误，仅必要时使用 |

颜色规则：

- 主要操作每个视区最多一个实心品牌色按钮；
- 普通卡片以细边框区分，不依赖大面积阴影；
- 暖棕色只用于书签、历史和少量创作提示；
- 红色只表达错误、删除或违规，不作为活动运营色；
- 标签使用浅底深字，不使用一排彩虹色。

### 字体与排版

界面字体采用系统中文字体栈，减少外部字体加载：

```css
font-family: Inter, "PingFang SC", "Microsoft YaHei", "Noto Sans CJK SC", sans-serif;
```

小说正文提供两类字体选项：

- 黑体：系统中文无衬线字体，默认；
- 宋体：`STSong`、`SimSun` 等本地衬线字体，适合长文阅读。

| 层级 | 桌面端 | 移动端 | 建议字重 |
| --- | --- | --- | --- |
| 页面主标题 | 28px | 24px | 600 |
| 区块标题 | 20px | 18px | 600 |
| 卡片标题 | 16px | 16px | 600 |
| 正文 UI | 14px | 14px | 400 |
| 辅助信息 | 12px | 12px | 400 |
| 阅读正文 | 18px | 17px | 400 |

阅读正文默认行高 `1.9`，段落间距 `1em`，桌面端正文最大宽度约 `760px`。

### 圆角、边框与阴影

- 小组件圆角：8px；
- 卡片与弹窗圆角：12px；
- 大型封面容器：16px；
- 默认边框：`1px solid #E6E8E2`；
- 普通卡片不使用阴影，悬停时只轻微上移并增加柔和阴影；
- 浮层阴影控制在低透明度，不使用黑色重投影。

## 5. 全局导航

### 桌面端顶部导航

```text
[若林 Logo]  首页  书库  排行  [搜索框........]  创作中心  [通知]  [头像]
```

- 高度约 64px，暖白半透明背景，下方细边框；
- 左侧是 Logo 和三个核心发现入口；
- 中部搜索框最大宽度约 360px；
- 右侧是创作中心、通知和用户菜单；
- 未登录时右侧显示“登录”和弱化的“注册”。

### 移动端导航

顶部只保留 Logo、搜索和用户入口，底部固定导航：

```text
首页 / 书库 / 书架 / 创作 / 我的
```

- 底部导航不超过五项；
- 阅读页隐藏常规导航，改用沉浸式阅读工具栏；
- 键盘弹出和编辑器页面可隐藏底部导航；
- 移动端所有主要触控区域至少约 44px。

## 6. 核心页面规划

### 6.1 首页 `/`

首页目标：让用户在短时间内找到一本愿意打开的小说，同时让老用户快速继续阅读。

桌面端布局：

```text
┌──────────────── 顶部导航 ────────────────┐
│  欢迎语/轻量品牌文案       继续阅读卡片   │
├──────────────────────────────────────────┤
│  编辑推荐                                 │
│  [主推荐大卡] [作品卡] [作品卡]           │
├──────────────────────────────────────────┤
│  最近更新                       查看全部   │
│  封面 书名 / 最新章节 / 作者 / 更新时间   │
│  封面 书名 / 最新章节 / 作者 / 更新时间   │
├──────────────────────────────────────────┤
│  新作发现（横向作品卡）                   │
├──────────────────────────────────────────┤
│  分类入口：奇幻 日常 恋爱 科幻 悬疑……     │
└──────────────── 页脚 ────────────────────┘
```

规则：

- 不使用自动轮播 Banner；主推荐保持静态、可读且不抢夺控制权；
- 登录用户优先展示一张“继续阅读”卡；
- 作品卡包含封面、书名、作者、一行简介和最多两个标签；
- 最近更新使用紧凑列表，不把所有内容都做成大卡片；
- 首页首屏不弹登录、签到或活动弹窗。

### 6.2 书库 `/novels`

目标：通过明确筛选找到作品。

页面结构：

- 页面标题和简短说明；
- 一级分类、二级分类、状态、字数范围、更新时间和标签筛选；
- 排序：综合、最近更新、收藏最多、新作；
- 桌面端采用 4～5 列作品卡网格；
- 移动端采用封面在左、信息在右的紧凑列表；
- 筛选条件显示成可单独移除的浅色 Chip；
- 空状态提供“清除筛选”，而不是只显示“暂无数据”。

分类筛选采用联动方式：先选择一级分类，再展示该分类下已启用的二级分类。URL 使用稳定 ID 或 slug 保存筛选条件，刷新、分享和浏览器前进后退后仍能恢复筛选状态。

桌面端可以显示左侧分类导航或顶部两级筛选区；移动端使用筛选抽屉。选择一级分类时可以查看该一级分类下的全部作品，不强制继续选择二级分类。

### 6.3 排行 `/rankings`

- 使用“热门 / 收藏增长 / 新作 / 完结佳作”页签；
- 排名、封面、书名、作者、简短指标是主体；
- 前三名只做低饱和数字或细线强调，不使用金银铜大徽章；
- 明确标注统计周期和更新时间；
- 排行是发现辅助，不在首页堆叠多个榜单。

### 6.4 搜索 `/search`

- 搜索框保持在页面顶部；
- 支持搜索小说名、作者名和标签；
- 搜索结果支持按一级分类和二级分类继续筛选；
- 结果分为“小说”和“用户”，默认小说；
- 提供历史搜索与热门建议，但不泄露其他用户行为；
- 输入防抖，按 Enter 立即搜索；
- URL 保留查询参数，支持分享和浏览器前进后退。

### 6.5 小说详情 `/novels/[novelId]`

目标：帮助用户判断作品是否适合自己，并快速开始或继续阅读。

```text
┌──────────────────────────────────────────┐
│ [封面]  书名                             │
│         作者 · 连载状态 · 字数 · 更新时间 │
│         标签 标签                        │
│         简介（可展开）                    │
│         [开始/继续阅读] [加入书架] [分享] │
├──────────────────────────────────────────┤
│ 作品公告（存在时显示）                    │
├──────────────────────────────────────────┤
│ 目录                         [正序/倒序]  │
│ 第一卷                                  │
│ 章节 1 / 章节 2 / 章节 3                 │
├──────────────────────────────────────────┤
│ 作品评论                                 │
└──────────────────────────────────────────┘
```

- 封面比例统一为 3:4；
- 主要按钮根据进度显示“开始阅读”或“继续阅读”；
- 作者名可以进入公开主页；
- 简介默认展示适当行数，过长时展开；
- 目录按卷折叠，清楚标记免费/锁定/最新/已读状态；
- 分享使用系统分享或复制链接，并生成正确的标题、简介和封面；
- 评论位于目录之后，避免干扰作品判断与阅读入口。
- 访客可以直接点击“开始阅读”，不能先要求登录；
- 访客点击加入书架、点赞或发表评论时，再展示登录引导。

### 6.6 章节阅读 `/novels/[novelId]/chapters/[chapterId]`

这是最重要的体验页面。

```text
┌──── 顶部轻工具栏（滚动时可隐藏） ────────┐
│ 返回作品  章节名         目录  设置  更多  │
├──────────────────────────────────────────┤
│                                          │
│              第 X 章 标题                │
│           更新时间 · 本章字数             │
│                                          │
│     正文正文正文……                       │
│                                          │
│     正文最大宽度约 760px                 │
│                                          │
├──────────────────────────────────────────┤
│       [上一章]   [目录]   [下一章]         │
│       作者的话（存在时）                  │
│       本章评论                            │
└──────────────────────────────────────────┘
```

阅读设置：

- 字号：小、默认、大，或有限范围滑杆；
- 行高：紧凑、舒适、宽松；
- 字体：黑体、宋体；
- 主题：纸白、暖杏、护眼、夜间；
- 页面宽度：窄、标准、宽；
- 设置本地即时生效，登录后同步至账户；
- 自动记录章节和阅读位置；
- 未登录时将阅读设置和进度保存在当前浏览器，登录后可以提示用户同步；
- 键盘左右方向键切章前需避免与编辑/输入冲突；
- 移动端点击正文中部显示/隐藏工具栏，不默认做模拟翻页动画；
- 不在正文中插入漂浮广告、运营弹窗或连续动画。

夜间主题推荐使用深灰绿而非纯黑：背景 `#1F2420`，正文 `#D9DED6`。

### 6.7 登录与注册 `/login`

- 单栏居中卡片，最大宽度约 400px；
- 支持项目最终确定的手机号/邮箱登录方式；
- 登录和注册用页签或明确链接切换；
- 用户协议和隐私政策靠近提交按钮；
- 错误展示在对应字段旁，不用 Toast 代替字段错误；
- 登录成功后返回原页面。
- 从点赞、评论或收藏触发登录时，应保存来源和操作意图，不把用户统一送回首页。

### 6.8 书架 `/bookshelf`

- 分为“正在阅读 / 想读 / 已读完”；
- 默认按最近阅读排序；
- 每项显示阅读进度、最新章节和更新时间；
- 有更新的作品用小圆点/浅色标记，不用大红角标；
- 支持批量管理，但默认界面保持简洁；
- 未登录时展示登录引导，不伪造空书架。

### 6.9 阅读历史 `/history`

- 按日期分组；
- 显示读到的章节和相对进度；
- 单项可删除，清空全部需要二次确认；
- 提供继续阅读按钮；
- 明确说明历史同步策略。

### 6.10 评论与通知

评论组件分为：

- 作品评论：讨论整部作品；
- 章节评论：讨论当前章节；
- 回复：最多展示有限层级，深层回复改为对话串；
- 操作：点赞、回复、举报、作者置顶；
- 剧透内容允许用户主动折叠标记。

评论列表公开可读；点赞、发表、回复、举报和管理自己的评论需要登录。前台账户区域提供“我的评论”，用户可以查看、跳转和删除自己的评论；是否允许编辑由后端规则决定。

通知中心分为回复、提及、作品更新和系统通知。使用未读圆点和分组，不把所有通知都做成红色数字角标。

### 6.11 用户公开主页 `/users/[userId]`

- 头像、昵称、简介和加入时间；
- 作者身份存在时展示其作品；
- 可选择公开的收藏/动态必须由用户授权；
- 不公开邮箱、手机号、阅读历史等隐私信息。

### 6.12 前台用户中心 `/account`

分区：

- 个人资料；
- 账户安全；
- 阅读偏好；
- 通知设置；
- 隐私设置。
- 我的评论；
- 我的举报与处理结果（后续）。

桌面端左侧分区导航，移动端使用分组列表进入二级页面。

### 6.13 前台创作中心 `/creator`

创作中心视觉沿用主站，只提高信息密度，不套传统管理后台皮肤。

创作中心是普通登录用户发表和管理自己小说的前台功能，不属于管理员后台。作者只能操作自己拥有或被授权协作的作品。

核心页面：

1. 我的作品：作品卡、状态、字数、最近更新和快捷操作；
2. 新建/编辑作品：封面、书名、简介、一级/二级分类、标签和连载状态；
3. 卷章管理：左侧卷章树，右侧操作区；
4. 章节编辑：标题、Tiptap 正文、字数、自动保存状态；
5. 发布确认：预览、作者的话、定时发布（后续）；
6. 草稿箱：最后保存时间、所属作品和恢复入口。

编辑器桌面布局：

```text
┌───────────┬──────────────────────────────┐
│ 卷章目录  │ 章节标题                     │
│           │ 编辑工具栏                   │
│ 第一卷    │                              │
│  第1章    │ 正文编辑区域                 │
│  第2章    │                              │
│           │                              │
├───────────┴──────────────────────────────┤
│ 已自动保存 · 2,356 字       预览  发布    │
└──────────────────────────────────────────┘
```

- 自动保存不打断输入；
- 离开未保存草稿时提示；
- 发布与保存草稿在视觉上明确区分；
- 移动端允许轻量编辑草稿，但复杂卷章管理优先保证平板和桌面端。

#### 小说分类选择

- 作者不能自行创建、改名或删除分类；
- 分类数据由管理员后台统一维护；
- 新建小说时先选择一个一级分类，再选择其下一个二级分类；
- 第一版每部小说固定一个主分类路径，即“一个一级分类 + 一个二级分类”；
- 标签用于补充题材、要素和氛围，可以多选，但不能替代分类；
- 编辑作品时如果管理员停用了原分类，已有作品仍显示原分类，作者必须在再次发布关键变更前选择新的有效分类；
- 分类选择器显示完整路径，例如“奇幻 / 异世界”，避免同名二级分类产生歧义；
- 分类加载失败时不得允许提交一个未知分类值。

### 6.14 管理后台 `/admin`

第一阶段只规划必要能力：全站内容审核、举报处理、用户封禁、分类与标签配置。后台与普通用户权限由后端严格隔离，任何普通用户和普通作者均不可进入。

后台可以使用侧边导航，但仍复用 Nuxt UI 和项目 Token，不引入第二套 UI 框架。

#### 分类管理 `/admin/categories`

分类管理采用固定两级树：

```text
一级分类：奇幻
├── 二级分类：异世界
├── 二级分类：魔法校园
└── 二级分类：东方奇幻
```

管理员可以：

- 创建、编辑、排序、启用或停用一级分类；
- 在一级分类下创建、编辑、排序、启用或停用二级分类；
- 查看各分类当前作品数量；
- 将二级分类移动到另一个一级分类，但操作前必须明确影响范围；
- 合并分类，并将原分类作品迁移到目标分类；
- 为分类维护名称、slug、简短说明和可选图标；
- 预览该分类在书库筛选中的展示效果。

安全与数据规则：

- 只支持两级，不允许创建第三级，避免分类树失控；
- 已有作品使用的分类默认不能直接硬删除；
- 常规下线使用“停用”，停用后不再供新作品选择，但已有作品关系保留；
- 删除空分类也需要二次确认；
- 合并、移动和批量迁移必须显示受影响作品数量并记录管理员操作日志；
- 分类名称可以修改，但作品关联使用不可变 ID，不能依赖名称；
- slug 在同级内唯一，变更 slug 时需要处理旧链接重定向或兼容；
- 前端只负责交互，分类层级、唯一性、引用完整性和管理员权限必须由后端校验。

#### 分类与标签的区别

| 项目 | 分类 | 标签 |
| --- | --- | --- |
| 结构 | 固定两级树 | 扁平或按标签组展示 |
| 数量 | 每部作品一个主分类路径 | 每部作品可以有多个 |
| 创建者 | 仅管理员 | 第一版仍由管理员维护，作者只能选择 |
| 用途 | 书库导航和主要搜索筛选 | 题材元素、人物关系、氛围等补充筛选 |
| 示例 | 奇幻 / 异世界 | 慢热、群像、冒险、无恋爱线 |

## 8. 响应式规则

| 范围 | 设计策略 |
| --- | --- |
| `< 640px` | 单列、底部导航、紧凑小说列表、全宽抽屉 |
| `640–1023px` | 双列/三列卡片、适度留白、折叠侧栏 |
| `≥ 1024px` | 居中内容容器、桌面导航、多列内容 |
| `≥ 1440px` | 增加外侧留白，不无限拉宽正文和卡片 |

- 全站普通内容最大宽度约 `1200px`；
- 阅读正文独立限制在约 `760px`；
- 表单主体通常限制在 `720px` 以内；
- 移动端优先调整信息层级，而不是机械缩小桌面布局；
- 所有页面必须覆盖空数据、加载、错误、无权限和超长文字状态。

## 9. 动效原则

- 普通过渡控制在约 150～220ms；
- 使用淡入、颜色变化和轻微位移；
- 尊重 `prefers-reduced-motion`；
- 禁止背景持续漂浮、视差滚动、卡片大幅翻转和正文翻页特效；
- 加载优先使用骨架屏或局部进度，不用全屏旋转动画阻塞页面。

## 10. 无障碍与可用性

- 正文和关键控件满足足够的颜色对比度；
- 键盘可以完成导航、弹窗关闭、表单和章节切换；
- 焦点状态清晰但不刺眼；
- 图标按钮必须带可访问名称和 Tooltip；
- 封面图片必须有包含书名的 `alt`；
- 不只依靠颜色表达更新、错误或选中状态；
- 表单错误与字段关联，提交失败后聚焦首个错误；
- 标题层级保持语义正确。

## 11. MVP 页面优先级

### P0：首个可用版本

1. 全局导航和响应式布局；
2. 首页；
3. 书库与搜索；
4. 小说详情和目录；
5. 章节阅读和阅读设置；
6. 登录注册；
7. 书架和阅读历史；
8. 作品/章节评论；
9. 创作中心的作品、卷章、编辑和草稿；
10. 管理员两级分类管理、基础通知、举报和内容审核入口。

### P1：稳定后增加

- 完结佳作和更丰富的榜单；
- 作者数据统计；
- 定时发布；
- 更细的通知与隐私设置；
- 评论剧透、作者置顶和提及体验；
- 个性化推荐，但必须提供可解释的推荐依据。

### 暂不建设

- 付费阅读和充值；
- 实时聊天室；
- 直播、视频流；
- 复杂论坛系统；
- AI 写作；
- 社交动态流；
- 重度等级、勋章和签到体系。

## 12. 第一阶段设计验收

先实现并评审以下三页：

```text
首页 → 小说详情 → 章节阅读
```

验收重点：

- 桌面端和手机端的信息层级是否自然；
- 色彩是否淡雅、可读且没有后台模板感；
- 小说卡片是否能快速传达作品信息；
- 详情页能否快速开始或继续阅读；
- 访客能否无阻碍完成首页到章节阅读的完整流程；
- 点赞、评论和收藏是否只在操作发生时请求登录；
- 正文宽度、字号、行高和主题是否舒适；
- 加载、空状态、错误和长标题是否处理完整；
- SSR 首屏、SEO 信息和分享内容是否正确。

这三页通过后，再沿用已经验证的设计系统扩展书库、评论、账户和创作中心。
